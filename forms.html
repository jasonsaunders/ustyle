Fields Github: _fields.scss

  • .us-field-required - Any required form field has this class to show an asterisk beside the label
  • .us-field-blocked - Blocked label styling
  • .us-field-inline - Inline label styling

Icon inputs Github: _input-backgrounds.scss

In addition to our basic inputs we've got some useful input backgrounds that are used across the site. Each input background is a base64 encoded (for IE9 purposes) SVG to allow scalability on retina devices. Any browser not supporting SVG has a PNG fallback.

  • .postcode - Used for form fields asking for a postcode (typically in hero's)
  • .email - Used for email inputs
  • .phone - Used when asking for a phone number
  • .pound - Used when asking for cost of something
  • .password - For password only fields
  • .kwh - Energy specific

Example

.postcode
.email
.phone
.pound
.password
.kwh

Radios and checkboxes Github: _radio-checkbox.scss

Our standard radio and checkbox styling. Webkit/Blink browsers display our font icon tick or a box shadow circle. Other browsers fallback to default styling.

Example

                 <label>Do you have a different billing address?</label>
                 <div class='us-field-toggle'>
                   <label>
                     <input checked='checked' class='us-form-input' name='null' type='radio'>Yes</input>
                   </label>
                   <label>
                     <input class='us-form-input' name='null' type='radio'>No</input>
                   </label>
                 </div>
               </div>
               <div class='us-field us-field-blocked'>
                 <label>How would you like to be contacted?</label>
                 <div class='us-field-toggle'>
                   <label>
                     <input checked='checked' class='us-form-input' name='null' type='checkbox'>Mobile</input>
                   </label>
                   <label>
                     <input class='us-form-input' name='null' type='checkbox'>Email</input>
                   </label>
                 </div>
               </div>

Selects Github: _select.scss

The .us-form-select is built with inline background-image SVGs for Webkit/Blink browsers that allow for Sass variable changes on the hover/normal colours. Firefox allows for .us-form-select wrap to be added on top of the select to replicate styling, whilst other browser fallback to native implementation. $select-base-color, $select-hover-color and $select-active-color allow for different colouring of the border/dropdown.

Example


               <span class='us-form-select-wrap'>
                 <select class='us-form-select'>
                   <option>A select menu</option>
                   <option>With lots of options</option>
                   <option>And a few more</option>
                 </select>
               </span>

Form validation Github: _validation.scss

Validation message and input styling provides a consistent look and feel to uStyle forms.

  • .us-validation--bordered - use this for any singular field validation or validation on backgrounds
  • .us-validation--success - Success styling for correct forms
  • .us-validation--error - Error styling (default) for incorrect data on forms
